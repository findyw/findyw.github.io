从Java的角度简单对比Typescript
发表于 2020-03-24| 更新于 2020-03-26
阅读量:

序

没那么详细，只是大概列出TS与Java相关语法的对比……

记得刚毕业那会，个人简历上写着精通Java，现在想想都觉得好笑，不知道当时面试官看到简历之后是怎么想的。如今，从事Java工作也有好些年了，对Java语言的了解，不多也不少吧。虽然这几年主要开发工作都是做后端，但是对前端还是一如既往的保持浓厚的兴趣，这不，最近想要开发个小游戏，学习了阮大神的ES6教程，然后就开始投入开发。一个函数、两个函数、三个函数……写着写着，怎么感觉好乱啊；写到最后，都不知道怎么下手了。仔细想想，都做开发好几年了，面向对象还不会吗？不信邪，来研究一波TypesScript，再重构下之前的代码。

由于我个人比较喜欢通过看视频的方式来学习，所以我去B站找了视频来看，感兴趣的大佬可以去瞅瞅，我个人感觉老师讲的挺好，对于有Java开发经验的同学来说，完整看完一遍教程，马上投入TS开发工作我觉得是没有问题的。接下来要写的，基本上也是按照视频的内容进行整理的，但是并没有那么全，只是稍微对比下，做个记录。

P5 权限修饰符

Java中的权限修饰符有4种，由大到小依次为：public、protected、[friendly]、private；public相当于公共的，任何地方都能访问到；protected表示为受保护的，当前类以及派生类中可访问；friendly就是默认不写（留空）时的权限，相同的包下能访问；private则是私有权限，只有本类中能访问；

TS中权限修饰符中有3中，由大到小依次为：public、protected、private；默认留空，不写权限修饰符的时候是public；与Java类型，protected是派生类能访问的；private则是本类中才能访问

P6 接口（interface）

所谓接口，可以理解为定义了一套标准；标准有多重要相比大家都知道了吧。举个例子，你出门在外，你的手机快没电了，你得找地方充电，这时候你发现自己没有带线，而且自己的手机也比较高端，独一无二，充电线用的既不是usb-c，也不是usb-a，反正是厂商自己的一套标准。这时候，标准的作用就体现出来了，假如厂商使用的是usb-c，或者usb-a等，那满大街都是，随便就能充了。接口使用关键字 interface来声明，和Java一样。

需求是这样的，比如谷歌的安卓系统中照相机接口；我们都知道，照相机大概有以下功能，拍照、录像、变焦、对焦……那谷歌作为指定标准公司，它可以这么设计接口
